The following problems are similar to what you might see on the AP CS A exam. Please answer each to the best of your ability.

  • 12
  • 2 ^ 12 = 4096. Because 4096 is larger than 3000, 12 passes will be more than enough to guarantee finding the value.

    14-2-13: You have an array values filled with 50 integers. Which of the following correctly produces a random index of values?
  • (int) (Math.random() + 1) * 50
  • This always returns 50, as Math.random() + 1 becomes 1 when it is downcast to an int.
  • (int) (Math.random() * 50) + 1
  • This calculates a random index between 1 and 50, but indexes of arrays start at 0 and end at array.length - 1.
  • (int) (Math.random() + 1 * 50)
  • This always returns 50, since 50 is added to Math.random() and Math.random() returns a value less than 1.
  • (int) Math.random() * 50
  • This always returns 0, since Math.random() * 50 returns a double that is less than 1. When a double that is less than 1 is downcast to an integer, it always becomes 0.
  • (int) (Math.random() * 50)
  • This correctly calculates a random index between 0 and 49 for ``values``.

  • [4, 6, 7, 8, 13, 29, 12]
  • This is what would happen if selection sort was used instead of insertion sort. Remember that selection sort only swaps two elements at a time, while insertion sort places elements into a pre-sorted array.

    14-2-18: Which statement is equivalent to !( (x > 7) && !(y < 12) )?
  • (x <= 7) && (y < 12)
  • !(A && B) is NOT equivalent to (!A && !B). The AND needs to be changed to an OR.
  • (x <= 7) || (y < 12)
  • Using DeMorgan's law, !(A && B) is equivalent to !A || !B. So, the negation of (x > 7) is (x <= 7), and the negation of !(y < 12) is (y < 12).
  • (x > 7) || (y >= 12)
  • !(A && B) is NOT equivalent to (A || B). A and B need to become !A && !B. Also, (y >= 12) is equivalent to !(y < 12).
  • (x > 7) && (y >= 12)
  • !(A && B) is NOT equivalent to (A && B). !(y < 12) and (y >=12) mean the same thing; changing this does not make the statement the opposite.
  • (x <= 7) || (y >= 12)
  • !(A && B) is NOT equivalent to (!A && B). Changing !(y < 12) to (y >= 12) does not negate the statement; these two are equivalent.

    14-2-20: Consider the method findMax, which uses sequential search to find the index of the largest value of an array. In which case would findMax not work properly?

{

  {

  • The first value in the array is equal to 0.
  • This will not present a problem, as the if-statement has not been met and the for-loop will simply continue to the second element.
  • Every value in the array is greater than 0.
  • If every value in the array is greater than 0, the method will work properly.
  • Every value in the array is less than 0.
